[1] Canny Edge Detection (엣지 감지)
================================================================

▶ canny_low (0 ~ 255, 정수)
  - Canny 알고리즘의 하한 임계값
  - 이 값 이하의 픽셀 기울기는 엣지가 아님
  - 너무 낮으면 그림자, 노면 균열 등 노이즈까지 검출됨

▶ canny_high (0 ~ 255, 정수)
  - Canny 알고리즘의 상한 임계값
  - 이 값 이상의 기울기는 무조건 엣지로 확정
  - 두 임계값 사이는 이미 확정된 엣지에 연결된 경우에만 엣지
  - 너무 높으면 화질이 낮거나 차선이 흐릴 때 아예 검출 안 됨
  - 권장 비율: canny_low : canny_high = 1:2 ~ 1:3 (예: 50 / 120)


================================================================
[2] ROI (Region of Interest, 관심 영역)
================================================================

카메라 이미지에서 차선만 볼 사다리꼴 영역을 정의합니다.
BEV 변환 전에 이 영역만 잘라서 처리합니다.

     (top_width)
       ┌──────┐  ← top_y 위치
      /        \
     /          \
    └────────────┘  ← bottom_y 위치
      (bottom_width, center_x 기준)

▶ top_y (0.0 ~ 1.0)
  - 사다리꼴 윗변의 y 위치 (이미지 전체 높이 대비 비율)
  - 작을수록 더 먼 곳(화면 위)까지 ROI에 포함
  - 너무 작으면 하늘이나 건물이 들어가 오검출 위험

▶ bottom_y (0.0 ~ 1.0)
  - 사다리꼴 아랫변의 y 위치
  - 클수록 차 바로 앞까지 포함
  - 너무 크면 자차 후드가 ROI에 포함됨

▶ top_width (0.0 ~ 1.0)
  - 사다리꼴 윗변의 너비 (이미지 전체 너비 대비 비율)
  - BEV 원근 왜곡 품질에 가장 크게 영향
  - 작을수록 멀리 있는 차선이 BEV 이미지에서 더 평행하게 펴짐
  - BEV가 S자/뒤틀림이 생기면 이 값을 조정

▶ bottom_width (0.0 ~ 1.0)
  - 사다리꼴 아랫변의 너비
  - 보통 1.0(화면 전체 폭) 또는 자차 폭에 맞게 줄임

▶ center_x (0.0 ~ 1.0)
  - 사다리꼴의 가로 중심 위치
  - 0.5 = 화면 정중앙
  - 카메라가 좌우로 치우쳐 있으면 조정


================================================================
[3] Sliding Window (슬라이딩 윈도우)
================================================================

BEV 이미지에서 차선 픽셀을 아래에서 위로 탐색하는 알고리즘입니다.

▶ n_windows (5 ~ 80, 정수)
  - 세로 방향으로 나누는 탐색 윈도우 개수
  - 많을수록 급커브나 복잡한 차선도 정밀하게 추적
  - 너무 많으면 처리 속도 느려짐

▶ sw_margin (5 ~ 200 px, 정수)
  - 각 윈도우의 좌우 탐색 반경 (픽셀)
  - 크면 급격히 꺾이는 차선도 추적 가능, 단 노이즈에 민감
  - 작으면 안정적이지만 급커브에서 차선을 놓칠 수 있음

▶ minpix (1 ~ 100, 정수)
  - 윈도우 내 픽셀이 몇 개 이상이어야 다음 윈도우 중심을 이동시킬지
  - 낮으면 희미한 차선도 따라가지만 노이즈에 흔들림
  - 높으면 안정적이지만 차선이 끊긴 구간에서 추적이 멈춤

▶ minpix_bottom (1 ~ 200, 정수)
  - 맨 아래(첫 번째) 윈도우에서의 최소 픽셀 수
  - 차선 진입점 판단에 사용
  - minpix와 동일하게 작동하지만 시작점에만 적용

▶ max_gap_windows (0 ~ 5, 정수)
  - 픽셀이 부족한 윈도우(빈 구간)를 연속으로 몇 개까지 건너뛸지
  - 3이면 3개 윈도우 연속 비어도 계속 추적
  - 너무 크면 차선이 끊긴 곳에서도 노이즈를 따라감

▶ sw_bottom_offset (0 ~ 100 px, 정수)
  - 슬라이딩 윈도우 시작 위치를 이미지 맨 아래에서 얼마나 위로 올릴지
  - 차량 후드, 노면 반사, 하단 노이즈 제거용
  - 값이 크면 차 바로 앞 차선을 무시

▶ minlanepix (50 ~ 2000, 정수)
  - 전체 윈도우 탐색 후 해당 방향(좌/우)의 총 픽셀 수가
    이 값 미만이면 차선 없음으로 판단
  - 낮으면 희미하거나 짧은 차선도 인식
  - 높으면 안정적이지만 차선을 자주 놓칠 수 있음


================================================================
[4] Lane Geometry (차선 기하 파라미터)
================================================================

▶ lane_width (0.5 ~ 8.0 m)
  - 실제 도로 차선의 폭 (미터 단위)
  - 한쪽 차선만 보일 때(싱글 레인 모드), 차선에서 얼마나 옆에
    경로를 만들지를 이 값으로 결정
  - 경로의 위치 = 차선 위치 + lane_width / 2
  - 예: 3.5m → 차선에서 1.75m 측면에 경로 생성 (차선 정중앙)
  - 실제 차선 폭과 비슷하게 맞추는 것이 적절

▶ smooth_alpha (0.0 ~ 1.0)
  - 차선 폴리핏(다항 곡선 피팅)의 시간적 스무딩 계수 (EMA)
  - 공식: smooth = alpha × 이전 + (1 - alpha) × 현재
  - 0.7 기본값: 이전 프레임 70%, 현재 30% 반영
  - 높을수록: 차선이 흔들려도 경로가 안정적 (반응은 느림)
  - 낮을수록: 새 검출을 빠르게 반영 (노이즈에 민감)

▶ bev_yaw_offset (-2.0 ~ +2.0 m)
  - 카메라가 정면을 정확히 향하지 않을 때의 원근 오차 보정값 (미터)
  - 한쪽 차선만 보일 때, 경로 상단(멀리)이 차선 방향으로
    서서히 기울어지도록 조정하는 파라미터
  - 화면 맨 아래(차 근처)에는 영향 없음, 위로 갈수록 보정량 증가
  - 양수: 차선에서 경로를 더 차선 쪽으로 당김
  - 음수: 차선에서 경로를 더 차선 반대쪽으로 밀어냄
  - 직선 도로에서 경로가 한쪽으로 쏠리면 이 값으로 보정

▶ width_alpha (0.0 ~ 1.0)
  - 양쪽 차선이 모두 보일 때 측정되는 차선 폭의 EMA 계수
  - 폭이 갑자기 이상하게 측정돼도 이 값으로 완충
  - 높을수록 폭 추정이 천천히 변해서 안정적

[5] Save Config
================================================================

▶ save_config (체크박스)
  - 체크하면 현재 RQT 슬라이더 값 전체를 파일에 영구 저장
  - 저장 위치:
      src/vision_pkg/src/config.py
      src/vision_pkg/cfg/Vision.cfg
  - 저장 후 자동으로 체크 해제됨
  - 저장된 값은 다음 노드 재시작 시 기본값으로 적용됨
